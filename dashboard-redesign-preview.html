3 arah redesign untuk dashboard user GitHire

Tujuannya bukan cuma mengganti warna. Redesign akan merapikan semua menu user dashboard supaya alurnya jelas: profil, job matching, skill gap, roadmap belajar, bookmark, dan status lamaran.

A. Career Command Center

Rekomendasi saya. Semua menu diarahkan ke pertanyaan user: saya cocok kerja apa, kurang skill apa, harus belajar apa, dan lamaran saya sampai mana.

B. Premium Bento Visual

Paling kuat untuk demo hackathon: card besar, score visual, gradient, dan layout asimetris. Risiko: bisa terasa ramai kalau semua menu dibuat terlalu dekoratif.

C. Clean SaaS Productivity

Paling rapi dan operasional. Cocok kalau dashboard dipakai berkali-kali: table jelas, filter kuat, layout padat, visual lebih tenang.

Plan redesign yang saya sarankan

  • Pakai arah A sebagai struktur utama: dashboard berbasis workflow karier.
  • Ambil polish dari B secukupnya untuk halaman overview dan score/match cards.
  • Pakai kerapian C untuk menu list-heavy seperti Browse Jobs, Lamaranku, dan My Roadmaps.

Menu yang akan dirapikan

  • Overview: ringkasan next action, profile readiness, top matches, roadmap aktif, dan lamaran.
  • Profil & Skill: status GitHub/CV, skill verified, skill manual, dan update profile.
  • Browse Jobs + Rekomendasi: filter lebih jelas, card job konsisten, match score mudah dibaca.
  • Skill Gap + Roadmap: skill prioritas, alasan gap, progress belajar, dan CTA yang tidak membingungkan.
  • My Roadmaps + Lamaranku + Account + Onboarding: states, empty states, loading, dan mobile layout diseragamkan.
